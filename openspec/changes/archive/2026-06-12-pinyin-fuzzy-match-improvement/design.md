## Context

当前用户信息查询skill的smartSearch()方法在进行拼音匹配时，直接将用户的输入（可能是中文）与客户的拼音字段进行比较。这导致谐音字无法正确匹配。

例如，当用户输入"陈玮"时：
- 精确匹配：找不到"陈玮"（因为数据库中是"陈伟"）
- 拼音匹配：用"陈玮"去匹配"chenwei"，找不到（因为"陈玮"不是拼音）

问题根源：拼音匹配逻辑没有将用户的中文输入转换成拼音。

## Goals / Non-Goals

**Goals:**
- 修改smartSearch()方法，在拼音匹配前先将用户输入转换成拼音
- 支持谐音字匹配（如"陈玮"匹配"陈伟"）
- 保持其他匹配逻辑不变

**Non-Goals:**
- 不修改PinyinMatcher工具类
- 不修改UserInfo模型
- 不增加新的查询方式

## Decisions

### 1. 拼音转换时机

**选择**: 在smartSearch()方法中，拼音匹配前进行转换

**理由**:
- 只在需要时进行转换，避免不必要的计算
- 保持代码清晰，转换逻辑集中在一处

**实现方式**:
```java
// 在拼音匹配前，将用户输入转换成拼音
String queryPinyin = PinyinMatcher.getPinyin(query);
List<UserInfo> pinyinMatches = getCustomersByPinyin(queryPinyin);
```

### 2. 匹配策略

**选择**: 先尝试精确匹配，失败后再进行拼音匹配

**理由**:
- 精确匹配优先，保证准确性
- 拼音匹配作为补充，提高召回率

### 3. 首字母匹配

**选择**: 同样先转换再匹配

**理由**:
- 保持一致性
- 支持中文输入的首字母匹配

## Risks / Trade-offs

**[风险] 性能影响** → 拼音转换开销很小，可接受

**[风险] 误匹配** → 拼音匹配可能返回多个结果，用户需要选择

**[权衡] 准确性 vs 召回率** → 通过拼音匹配提高召回率，可能牺牲一些准确性
