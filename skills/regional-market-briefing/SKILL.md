# 区域市场简报指南

## 概述

本技能为银行网点生成全面的区域市场简报，整合以下内容：
1. **网点信息** - 银行名称、网点名称、城市、区域、地址周边
2. **区域经济数据** - 区域GDP、人口、产业结构
3. **财经资讯** - 近期金融政策和市场动态
4. **银行保险业竞争** - 银行业和保险业同业网点活动和市场格局
5. **市场机会** - 可操作的洞察和建议

支持的银行：浦发银行、工商银行、建设银行、中国银行、农业银行、招商银行等。

## 重要说明：使用腾讯联网搜索工具

**必须使用 `tencent_web_search` 工具进行信息收集，不要创建单独的工具或脚本。**

本技能通过多次有针对性的联网搜索，将结果整合为结构化简报。

## 操作步骤

### 第一步：提取网点信息

从用户输入中解析以下信息：
- **银行名称**：如浦发银行、工商银行
- **城市**：如上海、北京
- **网点名称**：如制造局路支行、朝阳支行
- **区域**：如黄浦区、朝阳区（如已提及）

```
用户输入: 浦发银行上海制造局路支行的区域市场简报
提取结果: bank="浦发银行", city="上海", branch="制造局路支行"

用户输入: 中国工商银行北京市朝阳区支行的市场环境分析
提取结果: bank="中国工商银行", city="北京", district="朝阳区", branch="朝阳支行"
```

### 第二步：识别网点所属区域

根据网点名称和城市信息，推断网点所属的行政区划：

**常见网点-区域映射**：
- 上海制造局路支行 → 黄浦区
- 上海陆家浜路支行 → 黄浦区
- 上海南京东路支行 → 黄浦区
- 上海淮海中路支行 → 徐汇区
- 上海静安寺支行 → 静安区
- 北京朝阳支行 → 朝阳区
- 北京海淀支行 → 海淀区

**区域识别逻辑**：
1. 如果用户已明确指定区域（如"黄浦区"），直接使用
2. 如果用户未指定，根据网点名称中的关键词推断
3. 如果无法推断，使用城市作为搜索范围，并提示用户

### 第三步：通过联网搜索收集数据

使用 `tencent_web_search` 进行 **6 次并行搜索**：

#### 搜索1：网点地址周边信息（主要）
```
tencent_web_search(query="{branch} {district} 周边 商圈 社区 交通")
```
示例：`tencent_web_search(query="制造局路支行 黄浦区 周边 商圈 社区 交通")`

#### 搜索2：区域经济数据（主要）
```
tencent_web_search(query="{district} 经济数据 GDP 人口 产业结构")
```
示例：`tencent_web_search(query="黄浦区 经济数据 GDP 人口 产业结构")`

#### 搜索3：区域财经资讯（主要）
```
tencent_web_search(query="{district} 财经资讯 金融政策 市场动态")
```
示例：`tencent_web_search(query="黄浦区 财经资讯 金融政策 市场动态")`

#### 搜索4：银行保险业竞争分析（主要）
```
tencent_web_search(query="{district} 银行 保险 网点 竞争 同业")
```
示例：`tencent_web_search(query="黄浦区 银行 保险 网点 竞争 同业")`

#### 搜索5：区域潜在客户特征（主要）
```
tencent_web_search(query="{district} 企业主 高净值人群 客户特征")
```
示例：`tencent_web_search(query="黄浦区 企业主 高净值人群 客户特征")`

#### 搜索6：城市级补充信息
```
tencent_web_search(query="{city} 金融 保险 市场 动态 趋势")
```
示例：`tencent_web_search(query="上海 金融 保险 市场 动态 趋势")`

### 第四步：整合分析并生成简报

分析搜索结果，生成结构化的Markdown简报：

```markdown
# 区域市场简报：{bank}{branch}

## 📍 网点基础信息
- **银行**: {bank}
- **网点**: {branch}
- **城市**: {city}
- **区域**: {district}
- **地址周边**: {从搜索1中提取网点周边的商圈、社区、交通等信息}

## 📊 区域经济概况
{从搜索2中总结区域GDP、人口、产业结构等信息}

## 📰 区域财经资讯
{从搜索3中列出3-5条关键财经资讯}
1. {资讯1}
2. {资讯2}
3. {资讯3}

## 🏦 银行保险业竞争态势
{从搜索4中总结区域内银行和保险网点分布和竞争格局}
- 银行业：{描述}
- 保险业：{描述}

## 👥 潜在客户特征
{从搜索5中分析区域内潜在客户特征}
- 企业主分布：{描述}
- 高净值人群：{描述}
- 消费特点：{描述}

## 💼 保险业务机会
{基于区域信息，分析保险业务机会}
1. {机会1}：{描述}
2. {机会2}：{描述}
3. {机会3}：{描述}

## 💡 市场机会与建议
{基于所有数据，提供3-5条可操作的建议}
```

## 响应格式

简报应使用清晰的Markdown格式：
- 使用##标题分隔章节
- 使用项目符号列表
- 使用数字编号列表表示建议
- 添加数据来源说明
- 添加生成时间戳

## 提升简报质量的建议

1. **具体化**：使用搜索结果中的实际数据，而非泛泛而谈
2. **可操作化**：建议应具体且可执行
3. **平衡性**：同时呈现机会和风险
4. **简洁性**：聚焦最相关的信息
5. **来源标注**：说明关键数据的来源
6. **不展示中间过程**：不要向用户展示原始搜索结果，只呈现最终整合简报

## 数据局限性

- 搜索结果是实时的，可能会有变化
- 某些区域的数据可能较少
- 经济数据可能不是最新的
- 始终注明数据仅供参考
